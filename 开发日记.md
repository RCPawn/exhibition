

# 📔 白族非遗智能展示与管理平台 · 开发全纪实

## 阶段一：核心构建 —— 3D 数字化档案与工业级 UI 适配
**实现日期：2025.12 中旬**

*   **业务背景**：旨在建立非遗展品的数字双生档案，打破传统管理系统“表格堆砌”的廉价感。
*   **技术实现**：
    *   **3D 渲染集成**：基于 WebGL 技术栈集成 `<model-viewer>` 组件，通过参数调优（camera-orbit, exposure）实现展品的高保真展示。
    *   **工业级 UI 规范**：参考得物（Poizon）极简风格，采用黑白金配色及硬朗边框，自主设计了“三轴（Trinity）”视口锁定布局，解决了复杂交互页面在若依框架内的位移感。
    *   **渲染稳定性修复**：针对 Vue 标签页缓存机制导致的渲染死循环，创新性引入 `Key` 机制与生命周期钩子（onMounted/watch）强制组件重绘，彻底解决白屏及控制台 Lit 警告。

## 阶段二：交互闭环 —— 高并发统计逻辑与行为存证（长期方案）
**实现日期：2025.12 下旬**
*   **业务背景**：实现点赞、收藏、浏览量的闭环。要求状态持久化（刷新不丢失）且数据真实。
*   **技术实现**：
    *   **行为记录表设计**：新建 `heritage_user_action` 表，通过 `itemId + userId + type` 建立唯一索引，从物理层面规避重复刷票。
    *   **事务一致性保障**：在 Service 层利用 `@Transactional` 事务，同步操作“交互记录表”与“主表统计字段”，确保行为存证与汇总数据的强一致性。
    *   **高并发原子更新**：SQL 层面摒弃“查-改-存”模式，采用原子操作 `update ... set count = count + 1`，并配合 `CASE WHEN` 逻辑从数据库底层实现计数值非负校验，提升了系统的鲁棒性。
    *   **状态实时回显**：重构详情页加载逻辑，通过字段名兼容性处理（isLiked vs liked），实现了用户跨会话登录后的精准状态同步。

## 阶段三：数据大屏 —— 数字驾驶舱与全站可视化分析
**实现日期：2025.12 底**
*   **业务背景**：为管理员提供全量数据的实时监测能力。
*   **技术实现**：
    *   **数字化资源构成**：基于 ECharts 实现环形图，展示 3D 模型与图文资料的占比。
    *   **全站交互看板**：聚合点赞、收藏、评论三方数据，利用横向柱状图实时反馈社区热度。
    *   **中间舞台 HUD**：在驾驶舱中央区域嵌入 3D 核心模型，配合圆环呼吸动画，构建了沉浸式的数字孪生舞台。
    *   **SQL 聚合调优**：编写复杂 SQL 统计各分类数量及热门排行 Top 5，加入 `COALESCE` 处理空表边界条件，保障了统计接口的稳定性。

## 阶段四：社区生态 —— 评论系统与个人发布 (UGC)
**实现日期：2026.01.01**
*   **业务背景**：将平台从“单向展示”转变为“双向互动”，引入用户生成内容。
*   **技术实现**：
    *   **评论闭环**：实现评论发布、列表异步加载、头像自动映射功能。
    *   **我的发布 (UGC)**：在个人中心实现数据隔离（createBy 过滤），允许普通用户上传并管理自己的非遗作品。
    *   **长表单滚动隔离**：针对多维数据表单（富文本+多附件），设计了 `.dialog-scroll-wrapper` 滚动容器，解决了若依 Dialog 撑破视口的交互 Bug。
    *   **操作审计闭环**：在后端显式调用 `SecurityUtils` 注入操作人信息，补全了 `create_by` 字段的自动记录逻辑。

## 阶段五：体系升级 —— 分类层级化重构与传承人管理
**实现日期：今日（2026.01.03）**
*   **业务背景**：解决分类颗粒度过粗的问题（如：“传统技艺”大类无法精准对应到“扎染”小类），建立更专业的文化保护脉络。
*   **技术实现**：
    *   **树形结构建模**：重构 `heritage_category` 为 **Adjacency List（邻接表）模型**，引入 `parent_id` 实现无限级分类。
    *   **递归 DFS 算法**：在后端利用递归深度优先算法将扁平化记录转换为嵌套树结构，支撑前端级联选择器（Cascader）的渲染。
    *   **技艺统领模型**：确立了“以技艺分类为枢纽”的弱耦合模型。传承人与展品分别挂载在“小类”下，实现了业务逻辑的天然解耦。
    *   **统计向上聚合**：重写大屏统计 SQL，利用 `JOIN` 关联父类 ID 向上溯源，实现了小类展品数据自动归集至大类展示，平衡了宏观与微观的数据颗粒度。
